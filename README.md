# Secret HTML

Статическая HTML-страница с зашифрованным контентом.
Контент открывается только после того, как пользователь соберёт правильную
секретную фразу из двухбуквенных слогов на экране.

## Как это работает

1. На странице отображаются слоги (комбинации из 2 русских букв) в случайных
   позициях, с разным регистром, углом наклона и цветом.
2. Пользователь нажимает на слоги, составляя слово. Каждый нажатый слог
   становится прозрачным и добавляется к фразе внизу экрана.
3. После каждого нажатия вычисляется производный ключ из составленной фразы
   (1000 раундов хеширования) и сравнивается его CRC с хранимым в `secret.json`.
4. Если CRC совпал — контент расшифровывается и отображается на странице.
5. При успешной расшифровке фраза сохраняется в `sessionStorage`. При обновлении
   страницы контент показывается сразу без ребуса.
6. Кнопка «Зашифровать» стирает `sessionStorage` и возвращает ребус.

## Файлы

| Файл           | Описание                                 |
| -------------- | ---------------------------------------- |
| `index.html`   | Точка входа                              |
| `style.css`    | Стили (тёмная тема, хакерский стиль)     |
| `script.js`    | Логика ребуса, криптография, UI          |
| `secret.json`  | Массив комбинаций + зашифрованные данные |
| `generate.js`  | Скрипт генерации `secret.json`           |
| `input.html`   | Исходный HTML-контент для шифрования     |

## Генерация секрета

Требуется Node.js.

```shell
node generate.js <input-file> <secret-phrase> <кол-во комбинаций>
```

Параметры:

- `<input-file>` — путь к HTML-файлу с секретным контентом
- `<secret-phrase>` — секретная фраза из русских букв (чётная длина, 6–20 символов, без повторяющихся пар)
- `<кол-во комбинаций>` — сколько слогов отобразить на странице (минимум: кол-во пар в фразе + 2)

Пример:

```shell
node generate.js input_example.html ПРИВЕТ 20
```

Фраза `ПРИВЕТ` разбивается на пары `ПР + ИВ + ЕТ`. На странице будет 20 слогов:
3 правильных + 17 случайных обманок.

## Запуск

На http сервер нужно разместить результат (статичные файлы):

- index.html
- style.css
- script.js
- secret.json

## Криптография

- **Хеш**: 4 × djb2 с разными seed → 32 hex-символа (128 бит)
- **Key derivation**: 1000 раундов `hash(h + phrase)` → 16-байтный ключ
- **CRC**: хеш производного ключа (не самой фразы)
- **Шифрование**: RC4-drop3072 (первые 3072 байт потока отбрасываются)
- **Формат данных**: `secret.json` содержит `{ c: [...], d: "CRC32hex + base64" }`

Секретная фраза не хранится ни в коде, ни в `secret.json`. Единственный способ
получить контент — собрать правильную фразу.

## Формат secret.json

```json
{
    "c": ["ПР", "АБ", "ИВ", ...],
    "d": "2d6735301460ae9a...base64data..."
}
```

- `c` — массив двухбуквенных комбинаций (правильные + обманки, перемешаны)
- `d` — строка: первые 32 символа — CRC производного ключа, остальное — base64-кодированный шифротекст
