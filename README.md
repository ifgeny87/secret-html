# Secret HTML

Статическая HTML-страница с зашифрованным контентом.
Контент открывается только после того, как пользователь соберёт правильную
секретную фразу из двухбуквенных слогов на экране.

## Как это работает

1. На странице отображаются слоги (комбинации из 2 русских букв) в случайных
   позициях, с разным регистром, углом наклона и цветом.
2. Пользователь нажимает на слоги, составляя слово. Каждый нажатый слог
   становится прозрачным и добавляется к фразе внизу экрана.
3. После каждого нажатия вычисляется производный ключ из составленной фразы
   (1000 раундов хеширования) и сравнивается его CRC с одним из секретов в пазл-файле.
4. Если CRC совпал с одним из секретов — отображается именно тот контент, который зашифрован этой фразой.
5. При успешной расшифровке фраза сохраняется в `sessionStorage`. При обновлении
   страницы контент показывается сразу без ребуса.
6. Кнопка «Зашифровать» стирает `sessionStorage` и возвращает ребус.

## Генерация пазл-файла (несколько секретов)

Требуется Node.js.

```shell
node tools/encode.js <кол-во комбинаций> <выходной.json> <файл1> <фраза1> [<файл2> <фраза2> ...]
```

Параметры:

- `<кол-во комбинаций>` — сколько слогов на странице (минимум: все уникальные пары из всех фраз + 3)
- `<выходной.json>` — путь к создаваемому пазл-файлу
- пары `<файл>` `<фраза>` — для каждого секрета: HTML-файл и фраза (русские буквы, чётная длина, 6–20 символов, без повторяющихся пар; фразы не должны совпадать)

Комбинации общие для всех секретов, уникальные; при недостаточном количестве комбинаций скрипт завершится с ошибкой и укажет минимум.

Пример (один секрет):

```shell
node tools/encode.js 20 html-static/secret.json input/input_example.html ПРИВЕТ
```

Пример (два секрета):

```shell
node tools/encode.js 25 html-static/secret.json input/page1.html ПРИВЕТ input/page2.html СЕКРЕТ
```

## Расшифровка пазл-файла в файлы

Скрипт `decode.js` по паре «фраза — файл» находит нужный секрет и пишет контент в файл:

```shell
node tools/decode.js <пазл-файл.json> <фраза1> <выход1> [<фраза2> <выход2> ...]
```

- Если фраза не подходит ни к одному секрету — ошибка в stderr, выход с кодом 1.
- Для каждой пары (фраза, файл) контент расшифровывается и записывается в указанный файл (UTF-8).

Пример:

```shell
node tools/decode.js html-static/secret.json ПРИВЕТ output1.html СЕКРЕТ output2.html
```

## Криптография

- **Хеш**: 4 × djb2 с разными seed → 32 hex-символа (128 бит)
- **Key derivation**: 1000 раундов `hash(h + phrase)` → 16-байтный ключ
- **CRC**: хеш производного ключа (не самой фразы)
- **Шифрование**: RC4-drop3072 (первые 3072 байт потока отбрасываются)
- **Формат пазл-файла**: `c` — строка из склеенных комбинаций по 2 буквы; `items` — массив строк, в каждой строке в начале 32 hex-символа (хеш), далее base64-шифротекст.

Секретные фразы не хранятся в пазл-файле. Контент доступен только по правильной фразе; в одном пазле может быть несколько секретов с разными фразами.

## Формат пазл-файла

```json
{
    "c": "ПРАБИВЕТ...",
    "items": [
        "2d6735301460ae9a...base64...",
        "a1b2c3d4...base64..."
    ]
}
```

- `c` — одна строка: комбинации по 2 буквы склеены подряд (общие для всех секретов, уникальные)
- `items` — массив строк: в каждой строке сначала 32 hex-символа (хеш ключа), затем base64-шифротекст
